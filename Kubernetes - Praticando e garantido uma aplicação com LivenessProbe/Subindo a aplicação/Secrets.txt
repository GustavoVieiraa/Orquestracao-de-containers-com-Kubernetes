Nesta aula, o foco foi em como lidar com informações sensíveis, como senhas e usuários de banco de dados, de forma segura em aplicações Kubernetes, utilizando Secrets.

Aqui está um resumo dos principais pontos abordados:

    Necessidade de Criptografia: Dados confidenciais como senhas não devem ser armazenados de formaPlain text devido a riscos de segurança.

    Introdução aos Secrets: Secrets são objetos do Kubernetes projetados para armazenar pequenas quantidades de dados confidenciais, como senhas, tokens e chaves.

    Criação de um Secret: Foi demonstrado como criar um arquivo secrets.yaml para definir um Secret com os dados de usuário e senha do banco de dados.

    Codificação Base64: Para armazenar os dados de forma segura, é necessário codificá-los em Base64. O Kubernetes pode fazer isso automaticamente usando stringData em vez de data no arquivo secrets.yaml.

    Acessando os Secrets: Foi explicado como acessar as informações armazenadas no Secret dentro do arquivo app.yaml, utilizando secretKeyRef para referenciar o Secret e a chave desejada (por exemplo, DB_USER e DB_PASSWORD).

Em resumo, a aula ensinou como criar e utilizar Secrets no Kubernetes para proteger informações confidenciais, garantindo que sua aplicação acesse dados sensíveis de forma segura.
